# Validation gate

A **validation gate** is an explicit **go / no-go** checkpoint backed by **evidence tasks** (IDs like **V1–V12** in the East Tennessee package): *“have we proven X before we bet Y?”* **Execution gates** (**G1**, **G2**, …) are **phase** milestones—see [`Execution gate`](execution-gate.md) for contrast.

Evidence requirements and task IDs live in the **business plan** validation artifacts—not in this stub alone.
