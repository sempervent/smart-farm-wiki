# Validation backlog — assumptions to prove before major spend

## Purpose

**Gate** **lumpy** **CAPEX** **(fence**, **water**, **herd**, **equipment**, **telemetry** **at** **scale**)** **on** **evidence**, **not** **hope**. **Operational** **checklist** **layer** **on** [`East Tennessee two-site farm business plan — validation backlog`](east-tennessee-two-site-farm-business-plan-validation-backlog.md)—**same** **IDs** **where** **possible**.

## Key decisions supported

| Spend type | **Do not** proceed until… |
|------------|----------------------------|
| **Fence** **(full** **perimeter** **MVP)** | **G1** **rows** **green** |
| **Water** **development** | **Hydrology** / **rights** **rows** **green** |
| **Herd** **scale-up** | **Market** **+** **margin** **rows** **green** |
| **Telemetry** **beyond** **pilot** | **Instrumentation** **priority** **+** **manual** **proof** |
| **Equipment** **purchase** | **vs** **custom** **hire** **comparison** **documented** |
